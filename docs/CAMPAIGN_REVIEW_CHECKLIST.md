# Campaign Review Checklist

Use this checklist before a campaign moves beyond `proposed` or `review`.

## Scope

- [ ] Campaign purpose is specific and public-interest oriented.
- [ ] Campaign does not support trading account top-up.
- [ ] Campaign does not support margin-call or trading-loss recovery.
- [ ] Campaign does not promise financial return.
- [ ] Campaign does not create or promote a token.
- [ ] Campaign does not require private account credentials.

## Privacy

- [ ] Beneficiary privacy level is defined.
- [ ] No personal documents are committed to the repository.
- [ ] Beneficiary identifiers are redacted or pseudonymous.
- [ ] Proof material is stored outside the public repo or published only in redacted form.

## Wallets and custody

- [ ] No private keys, seed phrases, or recovery phrases are present.
- [ ] No wallet signing code is present.
- [ ] No auto-transfer code is present.
- [ ] No exchange withdrawal API is present.
- [ ] If wallets are needed, address-change procedure has been reviewed.
- [ ] Multisig or equivalent governance has been documented.

## Reporting

- [ ] Donation ledger columns are valid.
- [ ] Disbursement ledger columns are valid.
- [ ] Campaign metadata passes validation.
- [ ] Transparency report path is defined.
- [ ] Closeout report requirements are understood.

## Governance

- [ ] Conflict-of-interest disclosure is complete.
- [ ] Maintainer decision record is linked.
- [ ] Emergency freeze procedure is acknowledged.
- [ ] Donation activation gates remain inactive unless explicitly reviewed.

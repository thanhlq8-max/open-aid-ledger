# Roadmap

## v0.1 — public release foundation

- Publish public repository.
- Pass validation workflow.
- Keep donations inactive until wallet review is complete.
- Create labels and initial governance issues.

## v0.2 — wallet activation readiness

- Add reviewed `wallets.json`.
- Publish first wallet-change report.
- Add monthly report template.
- Add campaign status fields.

## v0.3 — transparency reporting

- Add generated monthly reports.
- Add asset-level balance summaries.
- Add manual reconciliation notes.
- Add privacy-safe beneficiary summary template.

## v0.4 — optional public chain readers

Only read public blockchain data. No private keys, no transfers, no exchange APIs.

Potential features:

- BTC address incoming transaction reader;
- EVM address incoming transaction reader through public explorer APIs;
- CSV reconciliation helper.

## v1.0 — stable public-good ledger

- Stable schema.
- Multisig recommendation implemented for active campaigns.
- Repeated monthly reports.
- Public contribution process validated.

## v0.3.0 — Sample Transparency Report + GitHub Pages Foundation

Status: implemented.

- Sample transparency report.

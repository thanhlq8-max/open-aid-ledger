# Donor FAQ

This FAQ answers common questions for future donors.

Current status:

```text
DONATIONS_ACTIVE: NO
WALLETS_PUBLISHED: NO
ACTIVATION_APPROVED: NO
```

This FAQ does not activate collection and does not publish live receiving details.

## Can I send support now?

No.

The public status still says support collection is inactive and receiving details are unpublished.

## How will I know when support is active?

The official public status page must clearly show:

```text
DONATIONS_ACTIVE: YES
WALLETS_PUBLISHED: YES
```

There must also be a reviewed receiving channel, exact asset, exact network, ledger policy, report policy, and freeze procedure.

## Where should I get the receiving detail?

Only from the official public status page or an approved repository source linked from it.

Do not rely on private messages, screenshots, forwarded text, or old copied information.

## Can I send from an exchange?

A future donor may use any compatible wallet or exchange that supports the selected asset and network.

The project should not require a specific exchange. Donors must check network compatibility before sending.

## What if I choose the wrong network?

Wrong-network transfers may be delayed, unrecoverable, or outside the project control.

The project must show the exact asset and network before any active release.

## How do I know support was recorded?

After reconciliation, the public ledger or transparency report should show a public-safe record.

The public record should not expose private details about supported people.

## Will donors receive profit, yield, repayment, or special access?

No.

Open Aid Ledger is not an investment product, trading fund, token project, repayment plan, or special-access program.

## How are supported people protected?

Public records should use case references and non-identifying summaries.

Private names, precise addresses, identity documents, contact details, and sensitive records should not be published in the public repository.

## What happens if something looks wrong?

The project should use the emergency freeze process.

A channel or report can be marked under review while maintainers preserve evidence, reconcile records, and document findings.

## What should I do today?

For now:

```text
DO_NOT_SEND: YES
READ_DONOR_QUICKSTART: YES
CHECK_PUBLIC_STATUS: YES
```

Wait for a separate reviewed activation proposal before sending any support.

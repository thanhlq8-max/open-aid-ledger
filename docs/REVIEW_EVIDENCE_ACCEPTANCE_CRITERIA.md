# Review Evidence Acceptance Criteria

## Purpose

This document defines minimum acceptance criteria for evidence used in an activation proposal.

## Evidence quality levels

| Level | Meaning |
|---|---|
| accepted | Sufficient evidence for current phase |
| partial | Useful but incomplete evidence |
| rejected | Evidence is missing, unverifiable, or unsafe |
| not applicable | Area does not apply to the current phase |

## Minimum criteria

Evidence must be:

- specific to this repository;
- linked to a reviewed file, checklist, or procedure;
- safe for public disclosure;
- free of private beneficiary details;
- free of wallet-control secret material;
- consistent with inactive donation status;
- reviewed by at least one maintainer.

## Activation blocker criteria

An activation proposal remains blocked if:

- any blocker finding is open;
- any high-severity finding is unresolved;
- wallet publication precheck is incomplete;
- beneficiary privacy review is incomplete;
- public status page review is incomplete;
- activation signoff record is missing;
- donation status is changed without reviewed evidence.

## RC3 boundary

RC3 accepts evidence for review. It does not activate donation collection.

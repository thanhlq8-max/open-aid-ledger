# Contributing

Contributions are welcome if they improve transparency, safety, reporting quality, documentation, or beneficiary privacy.

## Good contributions

- improve report generation;
- improve ledger validation;
- improve privacy-safe templates;
- improve documentation clarity;
- add read-only blockchain explorer importers;
- add tests;
- improve campaign reporting.

## Not accepted

- wallet signing;
- private-key handling;
- automated transfers;
- exchange trading APIs;
- trading signals;
- investment logic;
- token issuance;
- referral/airdrop engagement farming;
- beneficiary doxxing.

## Pull request rule

Every PR that touches donation policy, wallet metadata, or ledger records must explain:

1. what changed;
2. why it changed;
3. whether it affects fund custody;
4. whether it affects beneficiary privacy.
